## 🛠️ دليل تفعيل بوابة الدفع الحقيقية (Real Payment Gateway)

تم بناء النظام ليكون متوافقاً مع **PayMob** بشكل أساسي، ولكن يمكن تعديله لـ **Stripe** بسهولة.

### 1. الإعداد في PayMob / Stripe
*   قم بإنشاء حساب في بوابة الدفع.
*   احصل على الـ **API Key** و **Integration ID**.
*   قم بضبط الـ **Webhook URL** ليشير إلى: `https://your-server.com/api/payment/webhook`.

قم بتبديل كود الـ Mock في `create_checkout_session` بكود الاستدعاء الحقيقي:

```python
    # 1. الحصول على Token
    # 2. إنشاء Order
    # 3. إنشاء Payment Key مع metadata={"user_id": user_id, "plan_id": plan_id}

```

### 3. تأمين الـ Webhook
يجب تفعيل التحقق من الـ **HMAC Signature** في دالة `process_webhook` لضمان أن الطلبات قادمة من بوابة الدفع فقط وليس مخترقين.

### ⚠️ الأدوات الموجودة لكن تحتاج إكمالاً

#### 1. أداة القراءة السريعة (`ReadingSpeedTool.tsx` — 24KB)
**المشكلة:** الأداة ضخمة لكن لم يتم ربطها بنظام الاشتراك — أي أن القيود المقصودة غير مفعّلة.

**ما يجب فعله:**
- [ ] إضافة `useSubscription().checkAccess('fast_reading')` لتقييد الجلسات الطويلة
- [ ] إضافة حد للمستخدم المجاني (مثلاً: 5 دقائق فقط)
- [ ] إضافة رسالة ترقية عند الوصول للحد

#### 2. أداة التعليقات (`CommentsTool.tsx` — 41KB)
**المشكلة:** الأداة ضخمة، تحتاج مراجعة للتأكد من أن تصدير التعليقات لا يعطي خطأ غير متوقع عند ملفات كبيرة.

**ما يجب فعله:**
- [ ] اختبار مع ملفات PDF كبيرة (100+ صفحة)
- [ ] إضافة progress indicator أثناء الاستخراج

#### 3. إدارة الفهرس (`BookmarkTool.tsx` — 44KB)
**المشكلة:** الأكبر في الأدوات — ممتازة لكن قد يكون هناك edge cases في التقسيم التلقائي.

**ما يجب فعله:**
- [ ] اختبار التقسيم بالفهرس مع ملفات ذات تسلسل هرمي معقد

### المرحلة 2: بوابة الدفع الحقيقية

> [!WARNING]
> لا تنشر بدون دفع حقيقي إذا أردت تحقيق دخل

**الخيارات المتاحة في مصر:**

| البوابة | السهولة | الرسوم | التوصية |
|---|---|---|---|
| **Paymob** | متوسطة | 2.5% + ضريبة | ✅ الأفضل في مصر |
| **Fawry** | سهلة | رسوم ثابتة | ✅ شائع جداً |
| **Stripe** | سهلة جداً | 2.9% + $0.30 | ⚠️ للعملاء الدوليين فقط |
| **Gumroad** | سهلة جداً | 10% | ✅ خيار سريع جداً كبديل |

**التوصية الأسرع للنشر:**
استخدم **Gumroad** أو **LemonSqueezy** في البداية لأنها:
1. لا تحتاج تكامل كود معقد
2. تعطيك رابط دفع جاهز
3. تدير الضرائب والفواتير تلقائياً
4. عندك وقت لاحقاً للانتقال لـ Paymob

**كيف تعمل مع Gumroad:**
```
1. أنشئ منتج على gumroad.com
2. احصل على رابط الدفع
3. في payment_service.py، بدل Mock URL بـ Gumroad URL
4. استخدم Gumroad Webhooks لتفعيل الاشتراك
```
